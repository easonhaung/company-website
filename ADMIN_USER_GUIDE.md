# 📝 管理後台使用指南

## 🚀 快速開始

### 1. 登入系統
- **網址**: `http://localhost:3000/admin` 或從首頁右下角點擊 "Admin"
- **管理員帳號**: `admin` / `NewSecurePassword2024!`
- **編輯者帳號**: `editor` / `EditorSecurePass2024!`

### 2. 管理新聞
登入後，預設顯示「新聞管理」頁面，所有新聞按**新至舊**排序。

## 📰 發布新聞（無腦操作）

### 🎯 發布步驟
1. **點擊「新增新聞」** - 綠色按鈕
2. **填寫基本資訊**：
   - **標題**：新聞標題（必填）
   - **日期**：發布日期（必填，預設今天）
   - **摘要**：簡短描述（選填）
   - **圖片**：可上傳或選擇現有圖片（選填）
3. **撰寫內容**：
   - 使用 **Markdown 語法**
   - 支援 **粗體**、*斜體*、# 標題
   - 自動生成正確的 MDX 格式
4. **點擊「發布」** - 完成！

### 🖼️ 圖片管理

#### 📁 圖片上傳
1. **在編輯器中**：
   - 點擊「📁 上傳新圖片」
   - 選擇圖片檔案（支援 JPG、PNG、GIF、WebP）
   - 等待上傳完成（最大 5MB）

2. **在圖片管理頁面**：
   - 點擊右上角「🖼️ 圖片管理」
   - 批次上傳多張圖片
   - 查看所有現有圖片

#### 🎨 圖片選擇
- **點擊「選擇圖片」**：開啟圖片選擇器
- **網格顯示**：所有現有圖片縮圖
- **點擊選擇**：自動填入圖片路徑
- **手動輸入**：直接輸入圖片路徑

#### 📋 現有圖片導入
系統已自動導入 `public/news/` 目錄中的所有圖片：
- ✅ 新聞圖片
- ✅ 活動照片
- ✅ 產品圖片
- ✅ Logo 和其他素材

#### � 圖片路徑複製
- **在圖片管理頁面**：點擊圖片複製路徑
- **在編輯器中**：直接貼上路徑
- **格式範例**：`/news/image.jpg`

## 🛠️ 編輯現有新聞

1. **找到新聞**：在列表中找到要編輯的新聞
2. **點擊「編輯」**：藍色按鈕
3. **修改內容**：同新增新聞的介面
4. **點擊「更新」**：儲存變更

## 🗑️ 刪除新聞

1. **找到新聞**：在列表中找到要刪除的新聞
2. **點擊「刪除」**：紅色按鈕
3. **確認刪除**：系統會直接刪除檔案

## 📁 檔案管理

### 自動檔案命名
- **新聞檔案**：`YYYY-MM-DD-標題.mdx`
- **其他內容**：`標題.mdx`
- **儲存位置**：`content/news/` 目錄

### 圖片管理
- **圖片路徑**：相對於 `public` 目錄
- **儲存位置**：`public/news/` 目錄
- **支援格式**：jpg, png, gif, webp
- **大小限制**：單檔 5MB

## 🔧 進階功能

### Frontmatter 自動生成
系統會自動生成正確的 frontmatter：
```yaml
---
title: "新聞標題"
date: "2026-02-19"
excerpt: "新聞摘要"
type: "news"
image: "/news/image.jpg"
---
```

### 即時預覽
- 所有變更**立即生效**
- 無需重新啟動伺服器
- 可直接在網站上查看結果

## 🚨 注意事項

### ✅ 安全操作
- 所有操作都需要登入驗證
- 自動儲存操作記錄
- 支援多使用者同時操作

### ⚠️ 重要提醒
- **標題和日期為必填**
- **內容會直接寫入檔案**
- **刪除操作無法復原**
- **建議定期備份內容**

### 🎨 最佳實踐
- **標題簡潔明確**
- **摘要控制在 100 字內**
- **圖片使用相對路徑**
- **內容結構化，善用標題階層**
- **圖片壓縮優化**：建議寬度不超過 1200px

## 🆘 故障排除

### 常見問題
1. **無法登入**：檢查帳號密碼是否正確
2. **內容未顯示**：檢查日期格式是否正確
3. **圖片不顯示**：確認圖片路徑和檔案是否存在
4. **格式錯誤**：確保 Markdown 語法正確
5. **圖片上傳失敗**：檢查檔案格式和大小

### 圖片問題解決
- **圖片太大**：使用圖片編輯軟體壓縮
- **格式不支援**：轉換為 JPG 或 PNG
- **路徑錯誤**：使用 `/news/` 開頭的相對路徑

### 聯絡支援
如遇到問題，請聯繫系統管理員。
