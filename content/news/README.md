# 最新消息怎麼更新

所有「最新消息」都是 **MDX 檔案**，放在 `content/news/` 資料夾裡。不用後台、不用資料庫，直接新增或編輯檔案即可。

---

## 一、新增一則消息

### 1. 新增一個 `.mdx` 檔案

- **檔名**：建議用「日期-簡短標題」當檔名，用英文或數字，中間用 `-` 連接。  
  例如：`2026-05-15-new-partnership.mdx`
- **位置**：一定要放在 **`content/news/`** 底下。

### 2. 寫 frontmatter（檔案最上方）

在檔案**最上面**用 `---` 包起來，寫這幾欄：

```yaml
---
title: "這裡填消息標題（會顯示在列表與內頁）"
date: "2026-05-15"
excerpt: "這裡填一兩句摘要（會顯示在最新消息列表卡片下方）"
---
```

- **title**：必填，消息標題。
- **date**：必填，格式 `YYYY-MM-DD`，會用來排序（新的在上面）。
- **excerpt**：選填，列表頁會顯示的簡短說明。

若要區分類型，可以多一欄（目前列表不會顯示，之後可擴充）：

```yaml
type: "grant"   # 例如：grant / 合作 / 更新 / 公告
```

### 3. 寫內文

`---` 下面開始寫內容，用 **Markdown** 或 **MDX** 都可以：

- **粗體**：`**文字**`
- **列表**：`- 項目一`
- **連結**：`[文字](網址)`
- **圖片**：先把圖放到 `public/` 某個資料夾，再用  
  `![說明](/資料夾/檔名.png)`

範例內文：

```md
EZeeTech 於今日與 **某某單位** 簽署合作備忘錄。

重點包含：
- 技術驗證合作
- 試點計畫規劃

詳情請見 [官網連結](https://...)。
```

### 4. 存檔後重新建置

- **開發時**：若正在跑 `npm run dev`，存檔後重新整理頁面即可。
- **正式站**：改完後要重新執行 `npm run build` 再部署，新消息才會出現。

---

## 二、編輯或刪除消息

- **編輯**：直接改對應的 `.mdx` 檔案內容或 frontmatter，存檔後重新整理／重新 build。
- **刪除**：刪掉該則的 `.mdx` 檔案，重新 build 後，該則就不會再出現。

---

## 三、網址怎麼對應

- 檔名是 `2026-05-15-new-partnership.mdx` 時，**網址**就是：  
  `/news/2026-05-15-new-partnership`
- 所以檔名不要隨便改，否則舊連結會失效；若要改網址，等於要用新檔名新增一則，舊的可以刪掉或保留當紀錄。

---

## 四、圖片怎麼放

1. 把圖片放到專案裡的 `public/`，例如：`public/news/photo1.png`。
2. 在 MDX 裡用：`![圖片說明](/news/photo1.png)`。

這樣這則消息裡就會顯示該張圖。

### 兩張照片並排（統一尺寸）

若文章有**兩張照片**並排，請用 `<TwoPhotoGrid>` 元件，兩張圖會統一 4:3 比例、object-cover 填滿，並保留圖片說明：

```mdx
<TwoPhotoGrid
  leftSrc="/news/左圖.jpg"
  leftAlt="左圖替代文字"
  leftCaption="左圖說明"
  rightSrc="/news/右圖.jpg"
  rightAlt="右圖替代文字"
  rightCaption="右圖說明"
/>
```

可參考 `2025-10-29-SHINE-STAR-AWARD.mdx` 或 `2025-11-21-youth-competition-champion.mdx`。

---

有問題可以對照現有檔案，例如：`2026-04-30-u-start-pesa-01.mdx`。
